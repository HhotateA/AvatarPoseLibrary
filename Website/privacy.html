Avatar Pose Library プライバシーポリシー

最終更新日: 2026年7月23日

Avatar Pose Library（以下「APL」）は、利用状況の把握、品質改善、不具合調査のために、 Unity Editor上から次の情報を送信します。APLは、送信先が設定されていない場合や、 初回の選択が完了する前には情報を送信しません。

1. 送信モード

初回起動時に、次のいずれかを選択できます。選択内容は Tools > Avatar Pose Library > Privacy Settings から変更できます。

最小限のみ

  • APL専用のランダム識別子
  • APLバージョン
  • イベント種別（起動、バージョン変更、ビルド成功、ビルド失敗）

識別子はUnity Editorの設定領域に保存され、月間利用環境数を重複なく集計する目的で使用します。 VRChatアカウント、端末名、プロジェクト名、アバター名、ファイル名は送信しません。 識別子と許諾状態は Tools > Avatar Pose Library > Reset analytics and consent からリセットできます。

詳細送信を許可

最小限の情報に加えて、次の情報を送信します。

  • Unity、VRChat SDK、NDMFのバージョン
  • セッションID、ビルド時間
  • APLコンポーネント、ライブラリ、カテゴリ、ポーズの件数
  • Humanoid、Audio、Locomotion、FX、Cache、Auto Reset機能の利用有無
  • ビルド失敗時の処理段階と例外の種類

APLの処理中に捕捉した例外については、さらに次のエラーレポートを自動送信します。

  • 発生日時、APL専用ランダム識別子、各ソフトウェアのバージョン、Editorのプラットフォーム
  • ビルド統計と、APL設定値のうち機能の利用有無および件数
  • 例外の種類、エラー文字列、型名とメソッド名からなるスタックトレース

エラーレポートには、APLが収集項目として追加するプロジェクト名、アバター名、 オブジェクト名、アセットパスは含みません。ただし、外部ライブラリが生成した 例外メッセージ自体に、利用環境の情報やパスが含まれる可能性があります。

「最小限のみ」を選択中にAPLの例外が発生した場合は、設定値とエラー内容を送信するか 再確認します。同意すると、そのレポートを送信し、以後は「詳細送信を許可」として扱います。

2. 送信のタイミング

  • Unity EditorでAPLが読み込まれたとき
  • APLのバージョン変更を検出したとき
  • APLコンポーネントを含むアバターのビルドが成功または失敗したとき
  • APLの対象処理で例外を捕捉したとき（詳細送信のみ）

Unity Editor自体のクラッシュや、APLの対象外で発生した例外は収集しません。

3. 利用目的

  • 利用状況データは、APLの改善、利用傾向の把握、互換性の確認のために利用します。
  • 詳細なエラーレポートは、不具合の原因調査と修正のため、 運営者が管理する非公開の受信先へ送信します。
  • APLが送信するデータには、地域情報や広告識別子を含めず、 収集した情報を広告目的には利用しません。

4. 保管と第三者サービス

通常ログの保管と処理にはGoogle Analytics 4、エラーレポートの保管と処理には運営者が管理する非公開の受信先を利用します。 各サービスによる処理には、それぞれの利用規約およびプライバシーポリシーが適用されます。

運営者は、エラーレポートを不具合調査に必要な期間に限り保管し、 不要になったものから削除します。法令上必要な場合を除き、収集した情報を広告、 販売、または本ポリシーに記載のない目的で第三者へ提供しません。

5. お問い合わせ

本ポリシーおよび送信データに関する問い合わせは、 AvatarPoseLibraryのGitHub Issues から連絡してください。
